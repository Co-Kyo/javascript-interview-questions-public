## 题目背景

JavaScript 中 `Number` 类型采用 IEEE 754 双精度浮点数，安全整数范围为 `-(2^53 - 1)` 到 `2^53 - 1`，即 `Number.MIN_SAFE_INTEGER` (-9007199254740991) 到 `Number.MAX_SAFE_INTEGER` (9007199254740991)。

当整数超过这个范围时，`Number` 会丢失精度：

```js
console.log(9007199254740992 === 9007199254740993); // true ❌
console.log(9999999999999999999 + 1); // 10000000000000000000 ✅ (此例碰巧正确，但已超出安全范围，不保证可靠)
console.log(99999999999999999999 + 1); // 100000000000000000000 ❌ (精度丢失，实际应为 1000000000000000000000)
```

因此，对于超大整数的加法运算，不能直接使用 `Number` 类型，需要通过字符串模拟竖式加法来实现。

---

## 题目要求

实现一个函数 `add(a, b)`：

- **参数**：两个字符串 `a` 和 `b`，表示非负大整数（可能非常长）
- **返回值**：字符串，表示两个大整数的和
- **约束**：不使用 `BigInt`，必须通过字符串模拟竖式加法实现

---

## 示例

```js
add('999999999999999999', '1')
// → '1000000000000000000'

add('123', '456')
// → '579'

add('999', '1')
// → '1000'

add('0', '0')
// → '0'

add('1', '99999999999999999999999999999999999999999999')
// → '100000000000000000000000000000000000000000000'
```
