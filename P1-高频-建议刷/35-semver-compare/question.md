# 35 - 版本号比较 (Semver Compare)

## 分类
工程实用场景

## 难度
⭐⭐

## 考察点
- 字符串解析能力
- 逐段数值比较逻辑
- 对语义化版本（Semantic Versioning）规范的理解
- 边界情况处理

## 题目背景

在前端工程化中，版本号无处不在：

- **npm 依赖管理**：`package.json` 中 `"react": "^18.2.0"` 需要解析和比较版本号
- **升级检查**：CLI 工具需要判断当前版本是否落后于最新版本
- **依赖解析**：包管理器需要从多个可用版本中选择满足约束的最优版本
- **Changelog 生成**：需要按版本号排序展示变更历史

语义化版本（Semver）格式为 `MAJOR.MINOR.PATCH`，各段含义：
- **MAJOR**：不兼容的 API 变更
- **MINOR**：向下兼容的功能新增
- **PATCH**：向下兼容的问题修正

请实现一个版本号比较函数。

## 题目要求

实现 `compareVersions(v1, v2)` 函数，比较两个语义化版本号字符串：

- 当 `v1 > v2` 时返回 `1`
- 当 `v1 === v2` 时返回 `0`
- 当 `v1 < v2` 时返回 `-1`

### 基本约束
- 支持 `x.y.z` 格式（如 `1.2.3`）
- 支持不同长度的版本号（如 `1.2` 与 `1.2.0` 应视为相等）
- 各段应按**数值**而非**字符串**比较（`1.10.0 > 1.9.0`）
- 输入保证为合法的版本号字符串（非空、格式正确），无需处理无效输入

### Bonus
- 支持预发布标签比较：`alpha < beta < rc < 正式版`
  - 例如：`1.0.0-alpha.1 < 1.0.0-beta.1 < 1.0.0-rc.1 < 1.0.0`

## 示例

```js
compareVersions('1.2.3', '1.2.4')   // → -1  (patch 段 3 < 4)
compareVersions('2.0.0', '1.9.9')   // → 1   (major 段 2 > 1)
compareVersions('1.0.0', '1.0.0')   // → 0   (完全相等)
compareVersions('1.2', '1.2.0')     // → 0   (缺失段视为 0)
compareVersions('1.10.0', '1.9.0')  // → 1   (数值比较，10 > 9)
compareVersions('1.0.0-alpha', '1.0.0-beta')  // → -1 (bonus: alpha < beta)
```

## 提示

1. `String.prototype.split()` 可以按分隔符拆分版本号
2. 注意将字符串段转为数字再比较：`Number('10') > Number('9')`
3. 长度不同时，缺失的段应视为 `0`
4. 预发布标签通常以 `-` 分隔，如 `1.0.0-alpha.1`
