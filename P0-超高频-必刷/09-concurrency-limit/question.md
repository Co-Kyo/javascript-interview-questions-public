# 09 - 并发控制器（限制最大并发数）

## 基本信息

- **分类**：异步与并发控制
- **难度**：⭐⭐⭐
- **考察点**：Promise 池、任务调度、并发控制

---

## 题目背景

在前端开发中，我们经常需要同时发起大量异步请求，例如：

- 批量上传图片到服务器
- 批量调用接口获取数据
- 批量下载文件

如果不加限制地同时发起所有请求，会遇到以下问题：

1. **浏览器并发限制**：同域名下浏览器一般限制 6 个并发 TCP 连接，超出的请求会被排队等待
2. **服务器压力**：大量并发请求可能导致服务器过载、响应变慢甚至拒绝服务
3. **内存占用**：大量未完成的 Promise 同时存在，占用过多内存

因此，我们需要一个 **并发控制器**，限制同时运行的异步任务数量，当一个任务完成后自动从队列中取出下一个任务执行。

---

## 题目要求

实现一个 `Scheduler` 类，支持以下功能：

### 接口定义

```javascript
class Scheduler {
  constructor(max) {
    // max: 最大并发数
  }

  add(task) {
    // task: 一个返回 Promise 的函数
    // 返回一个 Promise，resolve 值为 task 的执行结果
  }
}
```

### 功能要求

1. `Scheduler` 构造函数接收一个参数 `max`，表示最多同时运行的任务数
2. `add(task)` 方法接收一个返回 Promise 的异步任务函数
3. 调用 `add` 后，如果当前运行的任务数小于 `max`，立即执行该任务
4. 如果当前运行的任务数已达 `max`，将任务放入等待队列，等有任务完成后再执行
5. `add` 方法返回一个 Promise，该 Promise 在对应任务完成时 resolve，resolve 值为任务的执行结果
6. 任务按添加顺序（先进先出）从队列中取出执行

### 示例

```javascript
const scheduler = new Scheduler(2);

const timeout = (time) => {
  return new Promise((resolve) => {
    setTimeout(resolve, time);
  });
};

const addTask = (time, order) => {
  scheduler
    .add(() => timeout(time))
    .then(() => console.log(`任务 ${order} 完成`));
};

addTask(1000, 1); // 任务 1：1s 后完成
addTask(500, 2);  // 任务 2：0.5s 后完成
addTask(300, 3);  // 任务 3：等待任务 2 完成后执行，0.3s 后完成
addTask(400, 4);  // 任务 4：等待任务 1 完成后执行，0.4s 后完成

// 输出顺序：
// 任务 2 完成（0.5s）
// 任务 3 完成（0.8s）
// 任务 1 完成（1.0s）
// 任务 4 完成（1.4s）
```

### 执行时序说明

以 `max=2` 为例：

| 时间点 | 正在执行 | 等待队列 | 事件 |
|--------|---------|---------|------|
| 0s | 任务1, 任务2 | [任务3, 任务4] | 任务1、2 立即执行 |
| 0.5s | 任务1, 任务3 | [任务4] | 任务2完成，任务3补上 |
| 0.8s | 任务1 | [任务4] | 任务3完成 |
| 1.0s | 任务4 | [] | 任务1完成，任务4补上 |
| 1.4s | - | [] | 任务4完成 |

---

## 约束条件

1. **不使用** 现成的并发控制库（如 `p-limit`、`p-queue` 等），需自己管理任务队列
2. 需要正确处理任务的 **错误情况**（某个任务失败不应阻塞后续任务）
3. 考虑 **边界情况**：max 为 1 时退化为串行执行，max 大于任务数时全部并行
4. **参数校验**：`max` 应为正整数，`task` 应为返回 Promise 的函数，不合法时应抛出有意义的错误
