# 31 - 分片上传（带进度）

## 基本信息

- **分类**：工程实用场景
- **难度**：⭐⭐⭐
- **考察点**：`File.slice`、并发控制、进度回调、断点续传

---

## 背景

大文件上传（网盘、视频平台、云存储）面临：网络波动导致重传、内存占用过高、无法精确展示进度、服务端请求体大小限制。需要**分片上传**方案：将大文件切割为多个小分片，并发上传，实时汇报进度，支持失败重试。

---

## 题目要求

实现 `chunkUpload(file, options)` 函数，支持：

1. **分片切割** — 使用 `File.slice()` 将文件切割为指定大小的分片（默认 1MB）
2. **并发控制** — 同一时刻上传的分片数不超过 `concurrency`（默认 3）
3. **进度回调** — `onProgress(percentage)` 回调，参数为 0~100 的百分比
4. **失败重试** — 单个分片失败时自动重试，支持指数退避（默认最多 3 次）
5. **完成回调** — 所有分片上传成功后调用服务端合并接口，再调用 `onComplete`
6. **取消支持** — 返回 `AbortController`，支持外部取消上传

## 约束条件

1. 使用 `File.slice()` 进行分片切割
2. 使用 `FormData` 上传，请求发往 `/upload` 接口
3. 并发控制不能使用 `Promise.all` 一次性发起所有请求
4. 重试时需有指数退避（exponential backoff）
5. 不依赖第三方库，纯原生 JavaScript 实现
