# 前端手写题库

> 35 道中大厂高频前端手写题，基于真实面经调研分级，每题配备完整学习材料。

## 这是什么

一个为 **3-5 年经验前端开发者** 设计的面试手写题备战库。不是又一份"前端面试题大全"，而是一套**可直接刷题的生产工具**：

- 35 道精选手写题，覆盖 JS 核心、异步并发、数据结构、DOM、设计模式、工程场景
- 每道题 **4 个文件**：题目描述 + 标准代码 + 逐层讲解 + 面经验证
- 基于真实面试数据标记优先级（P0/P1/P2），拒绝盲刷

## 为什么做这个

市面上的面试题库有两个问题：**太杂**（动辄几百题，不知道从哪开始）和**太浅**（只有题目没有讲解，看完还是不会写）。

这个题库的思路是：

1. **少而精** — 35 题覆盖 ~95% 的手写题考点，不凑数
2. **有据可查** — 每道题都标注了哪家公司、什么时间、在哪个平台出现过
3. **学到会为止** — 不只是给答案，而是从理解问题到写出代码到应对追问，一步步来

## 题库架构

```
frontend-interview-questions/
│
├── 🔴 P0-超高频-必刷/     15 题，覆盖面试 ~80%
│
├── 🟡 P1-高频-建议刷/     17 题，补到 ~95%
│
└── 🟢 P2-中频-选刷/        3 题，系统设计/项目深挖
```

每道题的目录结构：

```
01-call-apply-bind/
├── question.md           ← 题目 + 背景 + 示例 + 约束
├── solution.js           ← 标准答案 + 核心注释
├── explanation.md        ← 五步讲解（理解→思路→实现→追问→易错点）
└── interview-evidence.md ← 面经验证（公司/时间/来源/匹配度）
```

## 优先级说明

| 级别 | 数量 | 刷题耗时 | 含义 |
|------|------|----------|------|
| 🔴 P0 | 15 题 | 第 1 周 | 多公司多次出现，面试中几乎必考 |
| 🟡 P1 | 17 题 | 第 2 周 | 有明确大厂考察记录，出现频率较高 |
| 🟢 P2 | 3 题 | 第 3 周 | 偏系统设计或项目深挖，非高频编码题 |

优先级基于真实面经调研（覆盖字节/腾讯/阿里/美团/快手/百度/哈啰/支付宝等），数据来源包括牛客网、CSDN、掘金、知乎等平台。
